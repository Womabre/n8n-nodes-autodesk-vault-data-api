### Changelog

All notable changes to this project will be documented in this file. Dates are displayed in UTC.

Generated by [`auto-changelog`](https://github.com/CookPete/auto-changelog).

#### 1.1.0

- Add dynamic dropdowns (load from the API) for Role, User, Profile Attribute Definition and System Option fields
- Add vault-scoped dynamic dropdowns for Vault Option, Link and Change Order fields
- Replace the Folder field with a searchable Resource Locator ("From list" / "By ID"), backed by a recursive folder search within the selected vault
- Replace the File, Item and Markup ID fields with searchable Resource Locators ("From list" / "By ID")
- Replace the File Version and Item Version fields with a searchable Resource Locator listing the available versions, labelled with name, title and lifecycle state
- Remove redundant per-request `Authorization` headers; authentication is now handled solely by the credentials (fixes empty Bearer headers under OAuth2)
- Deduplicate the dynamic-dropdown routing definitions behind a shared helper
- Correct the node type, package namespace and node version in the node metadata (`node.json`)
- Clarify the Folder field description so it reads clearly in the UI and when the node is used as an AI tool
- **Breaking:** the Folder, File, File Version, Item, Item Version and Markup parameter values changed from a plain string to a resource-locator object (`{ mode, value }`); verify existing workflows that set these IDs still resolve after upgrading
- **Breaking:** "Get Many" operations now output one item per record (previously a single item containing the whole `{ pagination, results }` collection); update downstream nodes accordingly
